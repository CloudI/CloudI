Description

CloudI Work Manager

Queues up results that need to be sent to data modules, to avoid overloading the data repositories. This CloudI service can store a specific destination that is used for all binary traffic. Many destinations are handled in one work manager if any internal services (i.e., Erlang services, using the cloudi_service behavior) send "{Name, Data}" tuples to this service.
